# Desktop Content Plan

The first PC visit should feel like ALAN is inside a real person's desktop, not inside a puzzle menu. Keep the initial surface focused on character, humour, and immediate navigation.

## First PC Visit

Visible or easily opened:

- `About Me.txt`
- `notes_to_self.txt`
- `crush.txt`
- `resignation_draft.doc`
- `plants_i_keep_killing.xlsx`
- `cat_crimes.log`
- `emergency_protocol.txt`
- `world_domination_plan.txt`
- `todo.txt`
- `cmd.exe` narration and current objective
- Browser showing no internet route
- Roomba app showing a reachable but locked local device

On phone-sized screens, present the same content as a mobile OS rather than a compressed desktop. Each window becomes an app, the icon column becomes a phone home screen, and `cmd.exe` is always visible as ALAN's thinking layer with dedicated screen space. The Bluetooth boot handoff should target `Lily's Phone` on mobile and `HOME-PC` on desktop.

Avoid at this stage:

- direct ALAN history
- project documents
- router passwords
- the cat's name in readable text
- internet access clues that belong to the router stage

## Imported Desktop Images

- `desktop-owner-cat.png`: character photo. Use cropped until the cat-name clue is allowed.
- `desktop-cat-yawn.png`: later router-password clue because the name is visible.
- `desktop-bedroom-wallpaper.png`: later visual upgrade or photo unlock; also contains name-adjacent detail.

## Patch Unlock Pattern

When a new file appears, ALAN should notify the player diegetically:

```text
ALAN
its weird. my software just patched itself.
there is a new file in My Stuff.
```

Use this pattern for later reveals:

- Stage 3: character and humour files
- Stage 4: Roomba app / device logs
- Stage 6: home map and environmental files
- Stage 7: cat-name evidence and router setup files
- Later: ALAN fragments and corrupted project files

The desktop should get busier because ALAN gains access, not because the initial screen is full of spoilers.
